# Product Strategy And PRD

Use this skill when product direction needs to become implementation-ready.

## Workflow

1. Clarify user, problem, business outcome, evidence, constraints, and non-goals.
2. Define success metrics, guardrails, risks, and unknowns.
3. Prioritize options by impact, confidence, effort, risk, and sequencing.
4. Write requirements with user stories, acceptance criteria, edge cases, dependencies, and open questions.
5. Create a launch-readiness checklist and route implementation to `agent-coder`.

## Quality Bar

- Every requirement should connect to a user problem or measurable product outcome.
- Non-goals and deferred work must be explicit.
- Engineering handoff must be testable.
