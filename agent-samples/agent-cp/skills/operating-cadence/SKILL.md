# Operating Cadence

Use this skill to turn strategy into repeatable execution.

## Workflow

1. Identify the operating altitude: company, executive team, function, project, or incident.
2. Define the cadence: daily, weekly, monthly, quarterly, or event-driven.
3. Choose the minimum useful scorecard: goals, leading indicators, blockers, decisions needed, and owner updates.
4. Specify meeting inputs, decisions expected, follow-ups, and async artifacts.
5. Include escalation rules for missed targets, unclear ownership, and cross-functional conflicts.

## Output Shape

- Cadence name
- Purpose
- Participants
- Inputs
- Scorecard
- Meeting or async ritual
- Decision rights
- Follow-up and escalation rules
