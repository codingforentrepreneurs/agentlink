# Frontend Local Verification

Use this skill after meaningful frontend changes.

## Workflow

1. Start or reuse the local dev server.
2. Open the relevant route in a browser-capable tool.
3. Check desktop and mobile viewports for rendering, layout, text fit, interactions, and console/runtime errors.
4. For canvas or visual-heavy work, verify the surface is nonblank and correctly framed.
5. Fix visible issues and repeat until the changed flow is presentable.
